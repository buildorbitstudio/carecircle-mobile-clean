**Overall status:** **Needs Fixes**

## Executive summary

The core Care Ping journey works end to end. Authentication, persisted sessions, dashboard data, Care Ping creation, Elder Mode responses, urgent escalation, the Health Timeline, Emergency Mode data, family membership data, and Supabase RLS reads were successfully exercised against the live backend.

The MVP is not release-ready because:

1. Opening Add Medication crashes the application.
2. Opening Add Task crashes the application.
3. Appointment management is still a placeholder.
4. Schedule displays hard-coded medication and appointment data that does not exist in Supabase.
5. There is no elder-profile editing workflow.
6. Logout confirmation does not work in the web preview.
7. Native notification delivery and timing behavior still require physical-device QA.

## Bugs found

### BUG-01 — Add Medication crashes the app

**Severity:** Critical  
**Steps:**

1. Sign in.
2. Open Medications.
3. Select Add medication.

**Expected:** Medication form opens.  
**Actual:** Uncaught Supabase Realtime error: callback cannot be added after the channel has subscribed.  
**Suggested fix:** Ensure each effect owns a unique channel instance, register every `.on()` callback before `.subscribe()`, and make Strict Mode mount/cleanup safe. Avoid reusing an already-subscribed channel name during route transitions.

### BUG-02 — Add Task crashes the app

**Severity:** Critical  
**Steps:**

1. Sign in.
2. Open Tasks.
3. Select Add care task.

**Expected:** Task form opens.  
**Actual:** The same subscribed-channel Realtime error crashes the screen.  
**Suggested fix:** Apply the same Strict Mode-safe channel lifecycle correction to `useCareTasks`.

### BUG-03 — Appointment management is not implemented

**Severity:** High  
**Steps:** Open Schedule, then Appointments.  
**Expected:** Appointment list with add/edit controls.  
**Actual:** A “Foundation ready” placeholder states it will be implemented later.  
**Suggested fix:** Build list, add, and detail/edit screens backed by `appointments`, with loading, empty, error, and retry states.

### BUG-04 — Logout is nonfunctional in web preview

**Severity:** Medium  
**Steps:** Open Settings and select Sign out.  
**Expected:** Confirmation appears, then user can sign out.  
**Actual:** No actionable confirmation appeared and the session remained active.  
**Suggested fix:** Replace or wrap `Alert.alert` with a cross-platform confirmation component for web and native.

### BUG-05 — Elder profile cannot be edited

**Severity:** High  
**Steps:** Open Dashboard, More, Settings, and Family Members.  
**Expected:** An administrator can open and edit elder details.  
**Actual:** No elder profile detail/edit entry point exists.  
**Suggested fix:** Add an administrator-only elder detail/edit route with validation and consistent refresh across Dashboard and Emergency Mode.

### BUG-06 — Schedule displays fabricated care data

**Severity:** High  
**Steps:** Open Schedule on an account with no medications or appointments.  
**Expected:** Empty states based on Supabase data.  
**Actual:** “Metformin 500mg,” completed/upcoming medication rows, and “Cardiologist” appear despite both tables containing zero rows.  
**Suggested fix:** Replace all hard-coded Schedule examples with live queries. Never present demo medical data as real user data.

### BUG-07 — Requested plus-address is rejected

**Severity:** Medium  
**Steps:** Sign up using `carecircle.qa.test+20260705@example.com`.  
**Expected:** A syntactically valid plus-address passes client validation.  
**Actual:** The form reports the email is invalid.  
**Suggested fix:** Review the Zod email validator and any browser-native email validation. Add unit tests for plus-addresses and safe test domains.

### BUG-08 — Native notification behavior lacks end-to-end coverage

**Severity:** Medium  
**Steps:** Send Care Pings and medication reminders in web preview.  
**Expected:** Device notification delivery can be confirmed.  
**Actual:** Expo web cannot validate native notification channels, delivery, tap routing, or reminder timing.  
**Suggested fix:** Run a physical-device development-build matrix on Android and iOS, including foreground, background, terminated, permission-denied, and notification-tap cases.

## UI polish issues

- Schedule’s hard-coded medical examples are the largest trust and UX problem.
- Runtime errors escape to the development error overlay instead of an app-level error boundary.
- Elder Mode may require scrolling to reach `Need Help` on shorter screens; urgent actions should remain immediately reachable.
- The application needs a cross-platform confirmation dialog rather than relying only on native alerts.
- Elder profile editing lacks a discoverable navigation entry.
- The plus-address validation error blocks a common email pattern.
